## ğŸ—ï¸ Architecture

### Multi-Tenancy Model
- **Type**: Shared Database, Shared Schema
- **Isolation**: Row-level via tenant_id
- **Benefits**: Cost-effective, simple deployment
- **Scalability**: Can handle 1000+ tenants

### Authentication Flow
1. Register tenant with admin user
2. Login with email, password, subdomain
3. Receive JWT token (24-hour expiry)
4. Token in Authorization header
5. Auto-logout on expiry

### Role-Based Access
- **super_admin**: System-wide access
- **tenant_admin**: Organization management
- **user**: Limited team member access
